# MentalLog 要件定義書

## 1. 背景・目的

自分自身のメンタルケアができるようになることを目的とする。
その第一歩として、日々のメンタル状態を **ログ化（記録）** し、後から振り返れる状態をつくる。

### ゴール
- **ストレスの見える化**：日々のストレス・体力・メンタル余裕を数値で記録し、時系列で可視化する。
- **傾向と回復パターンの導出**：蓄積されたログから「何がストレス源になりやすいか」「どう回復してきたか」のパターンを見つけられるようにする。
- **思考のクセの自覚**：ストレス時に陥りがちな思考パターン（認知のクセ）を毎回チェックし、自己認知を促す（本アプリの最重要要素）。

### 非ゴール（初期スコープ外）
- 医療行為・診断の代替（あくまで自己記録ツール）。
- スマホネイティブアプリ化（初期はPC Webメイン）。
- 外部サービス（ウェアラブル、SNS等）連携。

---

## 2. 用語定義

| 用語 | 定義 |
|---|---|
| ログ | 1日1件のメンタル状態記録。数値・○×・テキスト・チェックから構成される。 |
| 数値項目 | ストレス・体力・メンタル余裕の0〜10スコア。 |
| ○×項目 | ストレス源となりうるカテゴリ（仕事・バンド関係など）。ユーザ毎にマスタ設定可能。 |
| チェック項目 | 「頭の中のクセ」「体の反応」「回復行動」の選択式チェックリスト。 |
| マスタ | ユーザが選択肢として使う項目の定義データ。 |

---

## 3. 想定ユーザ・ロール

| ロール | 説明 | 権限 |
|---|---|---|
| システム管理者 | アプリ全体の管理者 | 全ユーザ・全データの参照/管理、マスタ管理、ユーザ管理 |
| 一般ユーザ | 記録を行う本人 | **自分の情報のみ** 作成/参照/編集/削除。自分の○×項目マスタを設定可能 |

- 一般ユーザは他人のログを一切参照できない。
- 認証必須（ログインしないと利用不可）。

---

## 4. 機能要件

### 4.1 認証・ユーザ管理
- FR-01: ユーザはメールアドレス＋パスワードでログインできる。
- FR-02: ログイン/ログアウトができる。
- FR-03: システム管理者はユーザの登録・編集・無効化ができる。
- FR-04: 各ユーザはロール（管理者/一般）を持つ。

### 4.2 ログ記録
- FR-10: 一般ユーザは **1日1件** のログを作成できる（対象日を指定）。同一日の重複は不可（既存があれば編集）。
- FR-11: ログは以下で構成される。
  - **数値**（0〜10）：ストレス / 体力 / メンタル余裕
  - **○×項目**：ユーザのマスタに登録された項目に対し ○ / × を選択。○の場合は内容テキストを入力可能。
  - **テキスト**：「今日一番きつかったこと」「一言まとめ」
  - **チェック（頭の中のクセ）**：後述の選択肢から複数選択可（**超重要**）
  - **チェック（体の反応）**：複数選択可
  - **チェック（回復行動）**：複数選択可（できたものだけでOK）
- FR-12: ログの編集・削除ができる（本人のみ）。

### 4.3 ○×項目マスタ（ユーザ毎）
- FR-20: ユーザは自分の○×項目（ストレス源カテゴリ）を登録・編集・並び替え・無効化できる。
- FR-21: 初期値として以下を提供（テンプレート）：
  - 仕事
  - バンド関係
  - コミュニティ（複数登録可能）
  - 自分の疲労
  - その他
- FR-22: ○×項目で「○」を選んだ場合、内容を自由テキストで補足入力できる。

### 4.4 チェックリスト（マスタ）
以下3カテゴリの選択肢を提供する。初期はシステム共通マスタとして提供し、将来的にユーザ拡張を許容できる設計とする。

**頭の中のクセ（認知のクセ・超重要）**
- 全部ダメだと思った（0-100思考）
- 自分のせいだと思いすぎた
- 相手の気持ちを勝手に想像して疲れた
- 同時に全部解決しようとした
- 何も考えたくなくなった
- 特になし

**体の反応**
- 睡眠が浅い
- 胃・胸が重い
- イライラ
- 無気力
- 頭が回らない
- 特になし

**今日やった回復行動**
- 温泉・サウナ
- 食事で回復
- 音楽・バンド系
- 一人時間
- 軽い運動・散歩
- 何もできてない
- その他（テキスト入力）

- FR-30: 「その他」選択時はテキスト補足を入力できる。

### 4.5 ログ参照・分析
- FR-40: 一般ユーザは **自分のログのみ** 一覧・詳細参照できる。
- FR-41: 一覧の絞り込み条件：
  - 日付（期間 from/to）
  - 各数値の値（ストレス / 体力 / メンタル余裕）の範囲指定（下限・上限）
- FR-42: ストレス等の数値を時系列グラフで可視化できる（見える化）。
- FR-43: 傾向分析：○×項目・チェック項目の出現頻度を集計表示する（例：ストレスが高い日に多い思考のクセ）。
- FR-44: 回復パターン分析：回復行動と翌日以降のメンタル余裕/ストレス変化の相関を確認できる（初期は簡易集計でも可）。

### 4.6 管理者機能
- FR-50: システム管理者は全ユーザのログを参照できる。
- FR-51: システム管理者はチェックリストの共通マスタを管理できる。

## 5. 非機能要件

| 分類 | 要件 |
|---|---|
| 対象環境 | PC ブラウザメイン（Chrome/Edge/Safari 最新）。レスポンシブは将来対応。 |
| UI | HTML + CSS でモダンな見た目。入力のしやすさ最優先。 |
| 性能 | 個人利用規模。一覧は1000件程度でも快適に表示。 |
| セキュリティ | 認証必須。パスワードはハッシュ化。ユーザ間のデータ分離を厳格に（認可チェック必須）。 |
| プライバシー | メンタル情報という機微データを扱う。本人以外（管理者除く）非公開。 |
| 可用性 | 個人/小規模。特別なSLAは設けない。 |
| 保守性 | Laravel標準構成に準拠。マスタ拡張が容易な設計。 |

## 6. 制約・前提

- アーキテクチャ：**Laravel + PostgreSQL**、フロントは **HTML + CSS**（モダン、PCメイン）。詳細は `architecture.md`。
- 入力は日本語前提。日付・時刻はJST。
- 1日1ログを基本単位とする。

## 7. 画面一覧（概要）

詳細は `disp.md` を参照。

| 画面 | 概要 | 対象ロール |
|---|---|---|
| ログイン | 認証 | 全員 |
| ダッシュボード | 直近の状態・グラフ・当日ログ導線 | 一般/管理者 |
| ログ入力/編集 | 日次ログの作成・編集 | 一般 |
| ログ一覧 | 絞り込み・一覧表示 | 一般（管理者は全件） |
| ログ詳細 | 1件の詳細表示 | 本人/管理者 |
| 分析 | 傾向・回復パターンの可視化 | 一般/管理者 |
| ○×項目マスタ設定 | 自分のカテゴリ管理 | 一般 |
| ユーザ管理 | ユーザCRUD | 管理者 |
| チェックリストマスタ管理 | 共通マスタ管理 | 管理者 |
