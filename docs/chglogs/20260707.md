## 3. 本番デプロイ環境構築（UrlShare と同方式 / `mentallog.cde.jp`）
UrlShare の構成を踏襲しつつ、MentalLog は Blade+Vite（`public/build` が gitignore）である差異に対応。

- **`docker/php/Dockerfile` をマルチステージ化**：`base` / `development` / `build`（composer --no-dev・vite build・route/view cache）/ `production`（成果物焼き込み）/ `nginx`（Vite資産を焼き込み静的配信）。
  - named volume の陳腐化を避けるため、**Vite 資産を nginx イメージにも焼き込む**方式を採用。
- **`docker-compose.prod.yml`**：上位 nginx-proxy(jwilder)+letsencrypt で TLS 終端。`VIRTUAL_HOST=mentallog.cde.jp`、アプリ側はポート非公開。
- **`docker/nginx/default.prod.conf`**：`public/` 静的配信 + `php:9000` へFPMプロキシ。
- **`script/deploy.sh`**：VPS側で git reset→`up --build`→migrate→config/route/view cache。
- **`.github/workflows/deploy-product.yml`**：`product` への PR マージ or 手動実行で SSH 自動デプロイ。
- `.env.db(.example)` / `.env.production.example` / `.dockerignore` 整備。`.env.db` はサーバ認証情報保持のため gitignore。
- dev compose に `target: development` を明示（マルチステージ化に伴う）。
- 本番 php/nginx イメージのビルド検証（`public/build/manifest.json`・`vendor`・artisan 焼き込みを確認）。

## 5. 初回デプロイと障害対応
手動実行（workflow_dispatch）で初回デプロイ。以下を順に解消。
1. **deploy.sh パス不整合**：VPS の `product` チェックアウトが古くスクリプト未配置 → 最新化で解消。
2. **DB が SQLite にフォールバック**：本番 `.env` 未読込で `DB_CONNECTION` 既定=sqlite に。migration が `ALTER TABLE ... CHECK` でsqlite構文エラー → 本番 `.env` を正しく配置し pgsql 接続に。
3. **`APP_KEY` 未設定で HTTP 500**（`No application encryption key`）→ キー生成・設定し config 再キャッシュで解消。
- 結果：全マイグレーション適用、`GET /`→302→`/login`、`/login`→200、**Let's Encrypt 証明書（検証OK）で公開成功**。

## 6. 本番 HTTPS 認識の修正（TrustProxies）
- 上位プロキシが HTTPS 終端→内部 HTTP 転送のため Laravel が http と誤認し、`@vite`/`asset()` が `http://` URL を生成→CSS/JS が 404/mixed content。
- `bootstrap/app.php` に `trustProxies(at: '*')` を追加。dev で `X-Forwarded-Proto: https` によりアセットが https 化することを検証。
- 本番反映後、CSS/JS が `https://mentallog.cde.jp/build/assets/...` で 200 応答を確認。

## 7. bin スクリプトの本番対応 → 環境自動判定化
- 当初 `-p/--prod` フラグで本番 compose（`docker-compose.prod.yml`・サービス `php`）に切替対応。
- 「運用でオプションを付けたくない」との要望を受け、**`.env` の `APP_ENV` で dev/本番を自動判定**する方式に変更（オプション不要、`-p/-d` は任意上書き）。`production` 判定が壊れないことを検証。

## 8. 管理者作成コマンド
- `app/Console/Commands/MakeAdmin.php`（`app:make-admin {email} {--name} {--password}`）を追加。admin ロール・`is_active`・メール確認済みで作成/更新（`updateOrCreate`）。パスワードは省略時に秘匿入力。
- dev で動作検証（isAdmin=true / is_active / Observer による○×テンプレ5件生成）後、テストユーザ削除。
- 本番デプロイでイメージに焼き込み済み。VPS で `./bin/ssh.sh php artisan app:make-admin adm@cde.jp` により管理者投入可能な状態。

## 主な技術的判断
- 本番はコードをイメージに焼き込み、`.env`/`.env.db` は実行時にサーバ側で供給（gitignore + git reset で保持）。
- Blade+Vite の静的資産問題は、**nginx イメージにも Vite 資産を焼き込む**ことで named volume 陳腐化を回避。
- リバースプロキシ配下の https 認識は `TrustProxies` で対応（`forceScheme` ではなく正攻法）。
- 運用スクリプトは `APP_ENV` による環境自動判定とし、dev/本番で同一操作を実現。
