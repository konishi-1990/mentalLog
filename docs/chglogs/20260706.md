## 5. TDD 実装（全7フェーズ完了）

### フェーズ0：基盤＋テスト土台
- Laravel 導入。**Laravel 11 は 2026 時点でセキュリティ勧告によりブロック → 最新安定版 Laravel 13 を採用**。
- Breeze(Blade, --pest) 導入、`.env` を PostgreSQL 接続に設定。
- テストDB `mentallog_test` 用意（初期化スクリプト＋make target）、phpunit.xml を pgsql 化。
- サイドバー共通レイアウト＋CSSテーマ。最初のスモークテスト緑。
- npm キャッシュ権限問題を Dockerfile 修正で解消。

### フェーズ1：DB・モデル・認可基盤
- マイグレーション8テーブル（CHECK制約0-10・複合ユニーク）、Eloquentモデル8種。
- `UserObserver`：作成時に既定ロール付与＋○×テンプレ5件複製。
- シーダ（Role / ChecklistCategory / ChecklistOption）。

### フェーズ2：ログCRUD（コア・最重要 / MS2）
- `LogService::upsertDailyLog`（1日1件 upsert・子テーブル置換をトランザクション）。
- `Store/UpdateLogRequest`（0-10・「特になし」排他・requires_text）、`LogPolicy`（他人403）。
- `LogController` + 入力/編集/詳細/一覧ビュー（スライダー・○展開・排他JS）。
- ※実装中に `array_merge` 括弧漏れ（ParseError）をテストが検出→修正。

### フェーズ3：一覧・絞り込み
- `LogFilterRequest`、日付/各数値min/max、一般=自分のみ・管理者=全件＋ユーザ絞り込み。

### フェーズ4：○×項目マスタ設定
- `CheckItemController` + `CheckItemPolicy`（追加/改名/無効化/並び替え、他人操作403）。
- 無効化は論理削除で過去ログ保持。

### フェーズ5：見える化・分析（MS4）
- `AnalyticsService`：時系列 / ○×・チェック頻度（GROUP BY）/ 回復パターン（翌日メンタル余裕比較）。
- Chart.js 導入、ダッシュボード・分析画面。

### フェーズ6：管理者機能
- `EnsureUserIsAdmin` ミドルウェア、`Admin/UserController`・`Admin/ChecklistOptionController`。
- サイドバー管理メニューを isAdmin() で出し分け。

### フェーズ7：仕上げ（MS5 完成）

## 主な技術的判断
- Laravel 11→13 採用（勧告ブロック回避）。
- CSS は Breeze 同梱の Tailwind を採用（当初「素のCSS」から変更）。
- ○×テンプレ複製は Seeder ではなく **Observer** 方式（全作成経路で確実に複製）。
- CHECK制約は fluent 記法が無いため `DB::statement`。
- マスタ削除は物理削除せず無効化（参照整合性・過去ログ保持）。
